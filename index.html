    • peachfuzz

    Feeds garbage arguments and data into programs trying to induce a segmentation fault. Good for finding bugs.

    Project maintained by oxagast Hosted on GitHub Pages — Theme by mattgraham

    Intro

    peachfuzz, written primarily in C++, is designed to find code bugs by throwing garbage input at programs to see how they react. This is great for finding bugs, because not every type of input is always handled, and buffers are not always checked, etc. It also comes in handy when writing (and protecting against), buffer overflow exploitation.

    Checkout and compliation

    $ git clone https://github.com/oxagast/peachfuzz
$ cd peachfuzz
$ make
$ make test

    If you would like to play with faulty.c you can compile it with 

    $ gcc faulty.c -o faulty

    Usage

    The most simplistic usage is:

    $ ./peachfuzz -[tm] template_or_manpage -c /path/to/binary -b [buffersize]
$ ./peachfuzz -m mount -c /bin/mount -b 2048
$ cat filewithgarbagelinebyline | ./peachfuzz -t template ./faulty 2048

    Other

    Included is a file, bad_input.tar.xz. This has some pregenerated semi-psudorandom data in it. You can try it out by piping it to peachfuzz. You may want some more specific data to be piped in, if so, create a file line by line and pipe it in.
